LobeHub supports advanced voice capabilities, enabling natural voice conversations with AI agents through Text-to-Speech (TTS) and Speech-to-Text (STT) technologies.

## Overview

Voice features transform how you interact with AI:

- **Text-to-Speech (TTS)** - Hear AI responses read aloud with natural-sounding voices
- **Speech-to-Text (STT)** - Speak your messages instead of typing them
- **Voice conversations** - Interact with agents like you're talking to a real person
- **Multiple voice options** - Choose from diverse voices to match your preference
- **Multi-language support** - Works with various languages and accents

## Text-to-Speech (TTS)

TTS converts AI text responses into spoken audio, allowing you to listen instead of read.

### Enabling TTS

**For individual messages:**

1. Hover over an AI response
2. Click the speaker icon in the message actions
3. Audio playback begins automatically

**For automatic playback:**

- Configure agent settings to automatically read all responses
- TTS starts as soon as the AI completes its message

### Configuring Voice Settings

**Per-agent configuration:**

1. Open agent settings
2. Navigate to TTS section
3. Select voice provider (OpenAI or Microsoft)
4. Choose specific voice
5. Test with sample text
6. Save settings

Each agent can have its own voice, creating distinct personalities.

### TTS Playback Controls

When audio is playing:

- **Play/Pause** - Control playback
- **Progress bar** - See and seek through audio
- **Speed control** - Adjust playback speed (0.5x to 2x)
- **Volume** - Adjust audio level
- **Download** - Save audio file for offline use

### Audio Caching

TTS audio is automatically cached:

- **First playback** - Audio generates in real-time
- **Subsequent playback** - Instant playback from cache
- **Offline access** - Cached audio available without internet
- **Storage management** - Old cache automatically cleaned up

## Speech-to-Text (STT)

STT converts your spoken words into text, enabling voice input for messages.

### Using Voice Input

**To send a voice message:**

1. Click the microphone icon in the input area
2. Grant microphone permissions (first time only)
3. Speak your message clearly
4. Click the stop button when done
5. Review transcribed text
6. Edit if needed and send

**Tips for best results:**
- Speak clearly and at normal pace
- Use good microphone positioning
- Minimize background noise
- Speak in complete sentences
- Pause briefly between thoughts

### Supported Languages

STT supports multiple languages:

- English (US, UK, AU, CA, IN)
- Spanish (ES, MX, AR)
- French (FR, CA)
- German
- Italian
- Portuguese (BR, PT)
- Chinese (Mandarin)
- Japanese
- Korean
- And many more...

Language is typically auto-detected or set based on your interface language.

### Editing Transcriptions

After voice input:

1. Review the transcribed text
2. Click to edit any mistakes
3. Add punctuation if needed
4. Format or structure as desired
5. Send when ready

This hybrid approach combines the speed of voice with the precision of text editing.

## Voice Quality & Performance

### Audio Quality

**OpenAI Voices:**
- Sample rate: 24kHz
- Format: MP3, Opus, AAC, FLAC
- Bitrate: High quality encoding
- Latency: ~1-2 seconds

**Microsoft Edge Speech:**
- Sample rate: 16kHz - 48kHz
- Format: MP3, WAV, OGG
- Neural voices: Enhanced quality
- Latency: ~500ms - 2s

### Performance Tips

<AccordionGroup>
  <Accordion title="Optimize microphone setup">
    Use a quality microphone positioned properly for best STT accuracy. Reduce background noise and echo.
  </Accordion>

  <Accordion title="Check internet connection">
    Both TTS and STT require internet connectivity. Stable connection ensures smooth voice experiences.
  </Accordion>

  <Accordion title="Use appropriate voice speed">
    Adjust TTS playback speed based on content type. Slower for complex topics, faster for familiar content.
  </Accordion>

  <Accordion title="Cache for offline use">
    Download or cache important audio for offline access when internet may be unavailable.
  </Accordion>

  <Accordion title="Browser permissions">
    Grant necessary microphone permissions for STT. Check browser settings if voice input doesn't work.
  </Accordion>
</AccordionGroup>

## Privacy & Data

<Warning>
  Voice input is processed by AI services for transcription. Avoid speaking sensitive information unless using private/local deployments.
</Warning>

**Voice data handling:**
- STT audio sent to provider for transcription
- TTS audio cached locally for performance
- Audio not stored permanently by providers
- Transcriptions become part of conversation data

**Best practices:**
- Review transcriptions before sending
- Don't speak passwords or sensitive data
- Use private agents for confidential conversations
- Clear cache periodically if concerned about storage

<Note>
  Voice features require configuration of TTS/STT providers in settings. Some features may require API keys or subscriptions.
</Note>
