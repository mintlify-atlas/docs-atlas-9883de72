## Overview

The File API enables you to upload, manage, and organize files for use with AI agents and knowledge bases.

## Upload File

Create a file record after uploading to storage.

```typescript
const file = await client.file.createFile.mutate({
  name: 'document.pdf',
  fileType: 'application/pdf',
  size: 1024000,
  url: 'https://storage.example.com/files/document.pdf',
  hash: 'abc123...',
  knowledgeBaseId: 'kb_123', // optional
});
```

<ParamField path="name" type="string" required>
  File name
</ParamField>

<ParamField path="fileType" type="string" required>
  MIME type
</ParamField>

<ParamField path="size" type="number" required>
  File size in bytes
</ParamField>

<ParamField path="url" type="string" required>
  File URL from storage service
</ParamField>

<ParamField path="hash" type="string" required>
  File hash for deduplication
</ParamField>

<ParamField path="knowledgeBaseId" type="string" optional>
  Knowledge base ID to add file to
</ParamField>

<ParamField path="parentId" type="string" optional>
  Parent document/folder ID
</ParamField>

<ParamField path="metadata" type="object" optional>
  Additional metadata
</ParamField>

**Response:**

<ResponseField name="id" type="string">
  File ID
</ResponseField>

<ResponseField name="url" type="string">
  Proxy URL (format: `/f/{id}`)
</ResponseField>

## File Processing

### Chunking

When a file is uploaded, it's automatically chunked for RAG:

1. File is split into chunks based on content type
2. Chunking task is created with status 'pending'
3. Background worker processes the file
4. Status updates to 'processing', then 'success' or 'error'

### Embedding

After chunking, embeddings are generated:

1. Embedding task is created for each chunk
2. Background worker generates embeddings using configured model
3. Embeddings are stored for semantic search
4. Status updates to 'success' when complete

### Monitoring Status

Use `getFileItemById` to monitor processing status:

```typescript
const file = await client.file.getFileItemById.query({ id: 'file_123' });

if (file.chunkingStatus === 'error') {
  console.error('Chunking failed:', file.chunkingError);
}

if (file.finishEmbedding) {
  console.log('File ready for RAG with', file.chunkCount, 'chunks');
}
```

## File Storage

LobeHub supports multiple storage backends:

- **S3-compatible**: AWS S3, MinIO, Cloudflare R2
- **Local filesystem**: For development

Files are accessed via proxy URL (`/f/{id}`) which:
- Checks user permissions
- Redirects to actual storage URL
- Handles authentication

## Supported File Types

### Documents
- PDF (.pdf)
- Word (.doc, .docx)
- Text (.txt, .md)
- HTML (.html)

### Images
- JPEG (.jpg, .jpeg)
- PNG (.png)
- GIF (.gif)
- WebP (.webp)

### Code
- JavaScript (.js, .jsx)
- TypeScript (.ts, .tsx)
- Python (.py)
- JSON (.json)

### Other
- CSV (.csv)
- XML (.xml)
- YAML (.yml, .yaml)

Note: Image files are not included in knowledge base queries by default.
