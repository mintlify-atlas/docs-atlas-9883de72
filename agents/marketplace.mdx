# Agent Marketplace

The LobeHub Agent Marketplace is a vibrant community platform where you can discover, share, and collaborate on AI agents. Join a growing ecosystem of creators building the world's largest human-agent co-evolving network.

## Overview

The marketplace provides:

- **Discovery**: Browse 500+ community-created agents
- **Publishing**: Share your agents with the world
- **Versioning**: Track and manage agent updates
- **Analytics**: Monitor your agent's usage and installations
- **Collaboration**: Fork and improve existing agents

## Publishing Your Agents

### Preparation Checklist

Before publishing, ensure your agent has:

- [ ] Clear, descriptive title
- [ ] Comprehensive description (2-3 sentences)
- [ ] Appropriate avatar (emoji or image)
- [ ] Relevant tags (3-5 recommended)
- [ ] Well-tested system prompt
- [ ] Configured model and tools
- [ ] Opening message and questions (optional but recommended)

### Publishing Process

<Steps>

### Configure Agent

Create and refine your agent using the [Agent Builder](/agents/agent-builder):

```typescript
// Example well-configured agent
{
  meta: {
    title: "Python Code Reviewer",
    description: "Expert at reviewing Python code for bugs, style issues, and best practices. Follows PEP 8 and modern Python conventions.",
    avatar: "🐍",
    tags: ["python", "code-review", "development", "best-practices"],
    backgroundColor: "#3776ab"
  },
  config: {
    model: "claude-sonnet-4-5-20250929",
    provider: "anthropic",
    plugins: ["lobe-knowledge-base"],
    openingMessage: "I'm ready to review your Python code! Share your code and I'll provide detailed feedback.",
    openingQuestions: [
      "What aspect should I focus on? (bugs, style, performance)",
      "Is this production code or a learning project?"
    ]
  },
  systemRole: `You are a Python code review expert...`
}
```

### Test Thoroughly

Test your agent with various inputs:

- Typical use cases
- Edge cases and errors
- Different user skill levels
- Various input formats

### Publish to Marketplace

From your agent's profile page:

1. Click "Publish to Marketplace"
2. Review agent details
3. Add version notes (what's new, what changed)
4. Submit for publication

```typescript
// Agent is published with:
{
  marketIdentifier: "your-username/agent-name",
  version: "1.0.0",
  publishedAt: "2026-03-03T10:00:00Z",
  status: "published"
}
```

</Steps>

### Publishing Guidelines

<AccordionGroup>
  <Accordion title="Quality Standards">
    - **Clear Purpose**: Agent should have a specific, well-defined role
    - **Tested Behavior**: Consistent responses across similar inputs
    - **Good Documentation**: Clear description of what agent does
    - **Appropriate Model**: Model choice fits the use case
    - **Relevant Tools**: Only necessary tools enabled
  </Accordion>
  
  <Accordion title="Description Best Practices">
    - Start with what the agent does
    - Mention key capabilities or expertise
    - Indicate target users or use cases
    - Keep it concise (2-3 sentences)
    - Use clear, jargon-free language
  </Accordion>
  
  <Accordion title="Tag Selection">
    - Use 3-5 relevant tags
    - Include primary category (development, writing, etc.)
    - Add specific skills or domains
    - Use common, searchable terms
    - Avoid redundant or overly broad tags
  </Accordion>
  
  <Accordion title="System Prompt Quality">
    - Define role and expertise clearly
    - Include specific workflows or processes
    - Set communication style guidelines
    - Provide examples when helpful
    - Keep it focused and actionable
  </Accordion>
</AccordionGroup>

## Managing Published Agents

### Version Management

Update your published agents:

```typescript
// Create new version
{
  version: "1.1.0",
  changes: [
    "Added support for type hints review",
    "Improved async code analysis",
    "Updated to latest Python 3.12 features"
  ],
  publishedAt: "2026-03-15T10:00:00Z"
}
```

**Versioning Guidelines**:
- **Major (2.0.0)**: Breaking changes, complete redesign
- **Minor (1.1.0)**: New features, significant improvements
- **Patch (1.0.1)**: Bug fixes, minor tweaks

### Update Published Agent

<Steps>

### Modify Local Copy

Update your agent's configuration or prompt locally.

### Test Changes

Thoroughly test the updated version.

### Publish Update

From agent profile:
1. Click "Publish Update"
2. Enter version number
3. Describe changes
4. Submit

### Users Get Update Notice

Installed copies show update notification:
```
"New version available: 1.1.0
- Added support for type hints review
- Improved async code analysis"
```

</Steps>

### Unpublish Agent

Remove agent from marketplace:

```typescript
// Unpublished agents:
{
  status: "unpublished",
  unpublishedAt: "2026-03-20T10:00:00Z",
  reason: "major-revision" | "deprecated" | "merged"
}
```

**Note**: Unpublished agents can be republished later. Existing installations continue to work.

### Deprecate Agent

Permanently remove from marketplace:

```typescript
{
  status: "deprecated",
  deprecatedAt: "2026-03-25T10:00:00Z",
  message: "This agent has been replaced by 'Advanced Python Reviewer'"
}
```

**Warning**: Deprecation is permanent and cannot be reversed.

## Forking & Derivatives

### Fork an Agent

Create your own version based on existing agent:

<Steps>

### Install Base Agent

Install the agent you want to fork.

### Modify Configuration

Customize to your needs:
- Change model or parameters
- Adjust system prompt
- Add or remove tools
- Update metadata

### Publish as Derivative

When publishing, indicate it's based on another agent:

```typescript
{
  marketIdentifier: "your-username/custom-reviewer",
  forkedFrom: "original-author/python-reviewer",
  changes: "Specialized for data science code review"
}
```

</Steps>

**Attribution**: Derivative agents show their origin and credit the original creator.

### Derivative Best Practices

- **Add Value**: Make meaningful improvements or specializations
- **Credit Original**: Always acknowledge the base agent
- **Clear Differences**: Explain what's different/better
- **Respect License**: Follow original agent's terms
