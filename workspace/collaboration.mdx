# Team Collaboration

LobeHub's team collaboration features transform AI from a personal tool into a shared resource. Work together with teammates in a unified workspace where everyone can access agents, contribute to pages, and build on each other's work.

## What is a Team Workspace?

A team workspace is a shared environment where:

- **Multiple users** have access to the same agents and resources
- **Shared agents** are available to all team members
- **Collaborative pages** can be edited by multiple people
- **Projects** organize team initiatives
- **Visibility and permissions** ensure appropriate access

## Benefits of Team Collaboration

### Shared Context

Instead of everyone building their own agents:

- Team members use consistent agents with agreed-upon configurations
- Knowledge bases are centralized and accessible to all
- Everyone benefits from improved prompts and refinements

### Reduced Duplication

No more:

- Each person creating their own version of similar agents
- Copying and pasting agent configurations between team members
- Inconsistent agent behavior across the team

### Better Knowledge Management

- Centralized repositories of AI-generated insights
- Shared pages and documents everyone can access
- Institutional knowledge that persists beyond individual contributors

### Streamlined Workflows

- Standardized processes using shared agents
- Automated tasks benefit the entire team
- Clear ownership and accountability

## Team Workspace Features

### Shared Agents

Create agents that the entire team can use:

- **Company-specific agents**: Configured with your brand voice, product knowledge, and policies
- **Role-based agents**: Marketing analyst, technical writer, customer support assistant
- **Specialized agents**: Trained on internal knowledge bases and tools

#### Creating Shared Agents

<Steps>

### Create or Select an Agent

Build a new agent or choose an existing one from your personal workspace.

### Share with Team

In the agent settings, look for sharing or visibility options:

- Set visibility to "Team" or "Organization"
- Configure access permissions (view-only, edit, admin)

### Configure Team Settings

Optionally:

- Add the agent to specific teams or groups
- Set it as a default for certain workflows
- Link to team knowledge bases

</Steps>

#### Managing Shared Agents

For shared agents:

- **Ownership**: Assigned to a creator or admin
- **Editing**: Controlled by permissions
- **Visibility**: All team members can see and use the agent
- **Versioning**: Changes apply to all users

### Collaborative Pages

Pages can be shared across the team:

- Multiple team members can view and edit
- Real-time or near-real-time collaboration
- Version history tracks who made changes
- Comments and feedback (if supported)

#### Working Together on Pages

**Shared editing**:

- Multiple people can work on the same page
- Changes sync across team members
- Auto-save prevents conflicts

**Access levels**:

- **Viewers**: Can read but not edit
- **Editors**: Can modify content
- **Admins**: Can delete or change permissions

**Best practices for collaborative pages**:

- Communicate with teammates before major changes
- Use clear headings to claim sections you're working on
- Leave comments or notes for context
- Review changes regularly to stay aligned

### Team Projects

Projects provide structure for team initiatives:

- **Shared project space**: All team members see the same project view
- **Project members**: Assign specific people to specific projects
- **Roles and responsibilities**: Clear ownership within projects
- **Project dashboards**: Visibility into team progress

#### Setting Up a Team Project

<Steps>

### Create the Project

Set up a new project with a clear name and description.

### Add Team Members

Invite or assign team members to the project:

- Select from your organization's user list
- Assign roles (owner, contributor, viewer)

### Configure Project Resources

Add shared resources:

- Shared agents for the project
- Team knowledge bases
- Collaborative pages

### Set Permissions

Define who can:

- Create new pages and conversations
- Add or remove agents
- Invite new members
- Delete project resources

</Steps>

## User Roles and Permissions

Team workspaces typically support different user roles:

### Admin

- Full access to all workspace features
- Manage team members and permissions
- Create and delete shared resources
- Configure workspace settings

### Member

- Use shared agents and resources
- Create personal and shared content
- Collaborate on team projects
- Limited administrative capabilities

### Viewer

- Read-only access to shared resources
- Cannot create or modify agents
- Cannot edit shared pages
- Can view conversations and pages

### Guest (if supported)

- Temporary access to specific projects
- Limited to assigned resources
- Cannot access organization-wide features

## Team Management

### Inviting Team Members

Add people to your workspace:

<Steps>

### Access Team Settings

Navigate to workspace settings or team management.

### Send Invitations

Enter email addresses of people to invite:

- Choose their initial role
- Optionally add them to specific projects
- Send the invitation

### New Members Join

Invited users receive an email:

- They create an account or log in
- Accept the invitation
- Gain access to the team workspace

</Steps>

### Managing Team Members

Admins can:

- **Change roles**: Promote members to admins or adjust permissions
- **Remove members**: Revoke access when someone leaves
- **View activity**: See what team members are working on
- **Manage quotas**: Allocate usage limits per user (if applicable)

Learn more in [Managing Your Team](/usage/workspace/manage-team).

### Usage and Quotas

In team workspaces, usage may be tracked:

- **Organization-level quotas**: Total tokens or requests for the team
- **Per-user limits**: Individual usage caps
- **Project-based tracking**: Monitor usage by project

Learn more in [Managing Quotas](/usage/workspace/manage-quota).

## Collaboration Workflows

### Workflow 1: Shared Research

**Scenario**: Team researching a new market

1. **Create a team project**: "Market Research - Europe"
2. **Add a research agent**: Configured with relevant data sources
3. **Create collaborative pages**: Each team member drafts findings on separate pages
4. **Schedule automated updates**: Daily news summaries
5. **Weekly review**: Team discusses findings in a shared document

### Workflow 2: Content Production

**Scenario**: Marketing team producing blog content

1. **Shared agents**: Writer, editor, SEO specialist
2. **Content calendar page**: Tracks topics and deadlines
3. **Draft pages**: Writers create posts using the writing agent
4. **Review process**: Editors provide feedback on shared pages
5. **Publication**: Finalized content exported or published

### Workflow 3: Customer Support

**Scenario**: Support team handling customer inquiries

1. **Support agent**: Trained on product documentation and FAQs
2. **Shared knowledge base**: Product info, troubleshooting guides
3. **Case pages**: Each support case documented in a page
4. **Team visibility**: Everyone sees ongoing cases and resolutions
5. **Knowledge sharing**: Common issues documented for future reference

### Workflow 4: Product Development

**Scenario**: Product team planning a feature

1. **Project**: "Feature X - Design and Specification"
2. **Agents**: UX designer, technical architect, product manager
3. **Collaborative spec page**: Team iterates on requirements
4. **Scheduled tasks**: Weekly competitive analysis
5. **Decision log**: Track key decisions and rationale

## Best Practices for Team Collaboration

### Establish Naming Conventions

Consistent naming helps everyone navigate:

- **Projects**: "[Team] - [Initiative]" (e.g., "Marketing - Q1 Campaign")
- **Agents**: "[Function] - [Specialization]" (e.g., "Writer - Technical Docs")
- **Pages**: "[Project] - [Topic]" (e.g., "Q1 Campaign - Strategy Brief")

### Define Agent Ownership

Clarify who maintains shared agents:

- Assign an owner for each shared agent
- Owner is responsible for updates and improvements
- Changes to shared agents should be communicated to the team

### Use Projects for Structure

Don't let resources scatter:

- Organize all work into projects
- Make it clear which project each page and agent belongs to
- Regularly review and archive completed projects

### Communicate Changes

When modifying shared resources:

- Announce significant changes to agents or pages
- Explain why you're making the change
- Give team members time to adapt

### Regular Cleanup

Keep the workspace organized:

- Archive old projects monthly
- Remove unused agents
- Clean up outdated pages
- Review and update permissions

### Respect Permissions

Understand and honor access levels:

- Don't escalate permissions without cause
- Respect view-only access on sensitive projects
- Request access rather than working around restrictions

## Team Collaboration in Pages

### Simultaneous Editing

When multiple people edit the same page:

- LobeHub attempts to merge changes
- Conflicts are rare with auto-save and sync
- If a conflict occurs, the system may prompt you to review

### Comments and Feedback

If supported in your workspace:

- Leave comments on specific sections
- Tag teammates for input
- Resolve comments when addressed

### Version History

Track changes over time:

- See who edited what and when
- Revert to previous versions if needed
- Understand the evolution of a document

## Integrations for Team Collaboration

Depending on your workspace, you may integrate with:

- **Slack**: Post agent outputs to team channels
- **Microsoft Teams**: Share pages and notifications
- **Google Drive / OneDrive**: Sync pages to cloud storage
- **Project management tools**: Link LobeHub projects to Jira, Asana, etc.
- **Email**: Notifications for updates and changes

Check your workspace settings for available integrations.

## Privacy and Security

### Data Access

In team workspaces:

- Admins can access all team content
- Members can see shared resources and their own private work
- Private conversations and pages remain private unless explicitly shared

### Compliance

For regulated industries:

- Ensure your workspace meets compliance requirements (GDPR, HIPAA, etc.)
- Use appropriate access controls and data handling policies
- Review audit logs if available

### Sensitive Information

Best practices:

- Don't share confidential information in shared agents or pages without authorization
- Use private pages for sensitive work
- Review permissions regularly to ensure appropriate access

## Troubleshooting Team Collaboration

### Can't see shared resources

**Check permissions**:

- Ensure you have the right role
- Verify you're in the correct team or project

**Visibility settings**:

- Resource may be private or restricted
- Contact the owner to request access

### Conflicting edits on pages

**Wait for sync**:

- Give the system a moment to reconcile changes

**Communicate with teammates**:

- Let others know you're editing
- Work on different sections simultaneously

### Agent behaving inconsistently

**Check configuration**:

- Another team member may have modified the agent
- Review the agent's edit history if available

**Version differences**:

- Shared agents update for everyone
- If results differ, ensure you're using the same agent

## Scaling Team Collaboration

### Small Teams (2-10 people)

- Simple structure: shared agents and a few projects
- Informal communication about changes
- Everyone is an editor on most resources

### Medium Teams (10-50 people)

- Organized into sub-teams or departments
- Clear ownership of agents and projects
- More formal change communication
- Role-based permissions

### Large Organizations (50+ people)

- Multiple workspaces or organizational units
- Standardized naming conventions and governance
- Dedicated admins for workspace management
- Formal training and onboarding for new members
