# Projects

Projects in LobeHub help you organize your AI workspace by grouping related agents, conversations, pages, and resources. Whether you're managing a product launch, research initiative, or ongoing work stream, Projects keep everything structured and accessible.

## What is a Project?

A Project is a container that brings together:

- **Agents**: The AI assistants working on this project
- **Conversations**: Chat histories and ongoing discussions
- **Pages**: Documents and collaborative writing
- **Knowledge Bases**: Project-specific information and resources
- **Scheduled Tasks**: Automated agent runs related to the project

Think of a Project as a dedicated workspace for a specific goal or initiative.

## Why Use Projects?

### Before Projects

- Agents scattered across your workspace
- Lost context switching between unrelated conversations
- Difficulty finding past work on specific topics
- No clear boundaries between different initiatives

### With Projects

- **Focused context**: All related work in one place
- **Easy navigation**: Jump between project elements seamlessly
- **Better organization**: Clear separation of different workstreams
- **Team alignment**: Everyone sees the same project structure

## Creating a Project

<Steps>

### Access Projects

Navigate to your workspace and look for the "Projects" section in the sidebar or main navigation.

### Create New Project

Click "New Project" or the "+" button:

- Enter a **project name** (e.g., "Q2 Product Launch", "Research: Market Analysis")
- Add a **description** (optional) explaining the project's purpose
- Choose an **emoji or icon** (optional) for quick visual identification

### Set Up Project Structure

Once created, you can start adding:

- Existing agents or create new ones for this project
- Pages for documentation and planning
- Knowledge bases with project-specific resources

</Steps>

## Project Structure

### Organizing Agents

Add agents to your project based on their role:

- **Specialist agents**: Domain experts (e.g., Marketing Analyst, Technical Writer)
- **Generalist agents**: For broader tasks and coordination
- **Agent groups**: Teams of agents for complex workflows

Each project can have its own set of agents with project-specific configurations.

### Linking Pages

Connect pages to projects to keep documentation organized:

- Project briefs and requirements
- Meeting notes and decisions
- Research findings and analysis
- Final deliverables and reports

Pages automatically inherit project context, so agents understand the broader initiative.

### Managing Conversations

Conversations within a project maintain context:

- All chat history is associated with the project
- Agents can reference previous project conversations
- Easy to review the evolution of ideas and decisions

## Project Workflows

### Project-Based Navigation

Switch between projects to change your working context:

1. Select a project from the project list
2. Your workspace updates to show only project-related items
3. All new conversations and pages are automatically linked to the active project
4. Switch projects to move between different workstreams

### Cross-Project References

While projects keep work separated, you can:

- Reference pages from other projects
- Move agents between projects
- Link related projects together

### Project Templates

Create reusable project templates for common scenarios:

1. Set up a project with standard agents and structure
2. Save it as a template
3. Duplicate the template when starting new initiatives

Common template types:

- **Product Launch**: Marketing agent, technical writer, project manager
- **Content Creation**: Writer, editor, researcher, SEO specialist
- **Research Project**: Analyst, data agent, report writer
- **Software Development**: Code reviewer, documentation agent, QA assistant

## Project Management Features

### Project Dashboard

Each project has a dashboard showing:

- Recent activity and updates
- Active conversations and pages
- Project members and agents
- Upcoming scheduled tasks

### Search Within Projects

Find information scoped to your project:

- Search all pages and conversations in the project
- Filter by date, author, or content type
- Quickly locate past decisions and discussions

### Project Settings

Configure project-level settings:

- **Visibility**: Private or shared with team members
- **Access control**: Who can view or edit (in team workspaces)
- **Default agents**: Which agents to use by default
- **Knowledge base links**: Connect project-specific information

## Best Practices

### Keep Projects Focused

- One project per major initiative or goal
- Avoid overly broad projects like "Marketing" - break into specific campaigns
- Archive completed projects to keep your workspace clean

### Name Projects Clearly

Use descriptive names that communicate purpose at a glance:

- ✅ "Q2 2025 Product Launch - Mobile App"
- ✅ "Research: European Market Entry"
- ❌ "Project 1"
- ❌ "Stuff"

### Review Project Status Regularly

- Weekly: Review active projects and update priorities
- Monthly: Archive completed projects
- Quarterly: Clean up and reorganize as needed

### Use Projects for Accountability

In team settings:

- Assign project owners
- Set project milestones and deadlines
- Track progress through project dashboards

### Start Small

Don't create a complex project structure upfront:

1. Start with a simple project
2. Add agents and pages as needed
3. Let the structure evolve with your work

## Project Lifecycle

### Active Projects

Projects you're currently working on:

- Appear in your main workspace view
- Show in quick-access navigation
- Receive notifications for updates

### Archived Projects

Completed or paused projects:

- Removed from active view but still accessible
- All content preserved for reference
- Can be reactivated if needed

### Deleting Projects

When you delete a project:

- The project structure is removed
- Linked pages and conversations are preserved (unless explicitly deleted)
- Agents remain in your workspace
- Consider archiving instead of deleting for reference

- **Shared context**: Everyone on the team sees the same project view

- **Access control**: Manage who can view or edit project resources

## Tips for Effective Project Organization

1. **Use emoji or icons**: Visual markers make projects easy to spot
2. **Add descriptions**: Future you will appreciate context on older projects
3. **Regular cleanup**: Archive completed work monthly
4. **Consistent naming**: Develop a naming convention and stick to it
5. **Don't over-organize**: If a project feels empty, you might not need it

## Common Patterns

### Client-Based Projects

Agencies and consultants often organize by client:

- **Project per client**: "Client: Acme Corp"
- **Sub-projects per engagement**: "Acme - Website Redesign", "Acme - Content Strategy"

### Time-Based Projects

For ongoing work, organize by time period:

- "Q1 2025 Marketing"
- "January Newsletter"
- "2025 Annual Report"

### Theme-Based Projects

For research or exploration:

- "AI in Healthcare Research"
- "Sustainability Initiatives"
- "Learning: Python Development"

## Troubleshooting

### Project is too cluttered

- Break into smaller, focused sub-projects
- Archive old conversations and pages
- Review and remove inactive agents

### Can't find project content

- Use project-scoped search
- Check if items are linked to a different project
- Review archived projects

### Project performance is slow

- Large projects with many pages may take time to load
- Consider archiving old content
- Split into multiple smaller projects
