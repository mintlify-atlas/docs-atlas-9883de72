# Observability & Monitoring

LobeHub provides built-in observability features to help you monitor performance, debug issues, and understand user behavior in your self-hosted deployment.

## Monitoring Solutions

### Langfuse Integration

LobeHub integrates with [Langfuse](https://langfuse.com) for LLM observability, providing:

- Conversation trace logging
- Token usage tracking
- Cost analysis
- Performance metrics
- Model comparison
- User analytics

#### Configuration

<ParamField path="ENABLE_LANGFUSE" type="boolean" default="false">
  Set to `1` to enable Langfuse integration
</ParamField>

<ParamField path="LANGFUSE_PUBLIC_KEY" type="string" required>
  Langfuse public API key
  
  Get from [Langfuse Dashboard](https://cloud.langfuse.com) → Settings → API Keys
</ParamField>

<ParamField path="LANGFUSE_SECRET_KEY" type="string" required>
  Langfuse secret API key
  
  <Warning>Keep this secret secure. Never expose in client-side code.</Warning>
</ParamField>

<ParamField path="LANGFUSE_HOST" type="string" default="https://cloud.langfuse.com">
  Langfuse instance URL
  
  Use default for Langfuse Cloud, or your self-hosted instance URL.
</ParamField>

#### Setup Guide

1. Create account at [Langfuse Cloud](https://cloud.langfuse.com) or deploy [self-hosted Langfuse](https://langfuse.com/docs/deployment/self-host)
2. Create a new project
3. Navigate to Settings → API Keys
4. Copy public and secret keys
5. Configure environment variables:

```bash
ENABLE_LANGFUSE=1
LANGFUSE_PUBLIC_KEY=pk-lf-xxxxxxxx
LANGFUSE_SECRET_KEY=sk-lf-xxxxxxxx
LANGFUSE_HOST=https://cloud.langfuse.com
```

#### What Gets Tracked

When Langfuse is enabled, LobeHub tracks:

- **Traces**: Complete conversation flows
- **Spans**: Individual LLM calls, tool executions
- **Events**: User interactions, errors
- **Metadata**: Model names, token counts, latencies
- **Costs**: Token-based cost calculations

#### User Privacy

LobeHub respects user privacy settings:

- Users can opt-out via settings (controlled by `telemetry` preference)
- Only aggregated data is sent when opted-in
- Sensitive information is not logged

### OpenTelemetry Support

LobeHub includes experimental OpenTelemetry instrumentation for advanced observability.

#### Tracing

The application uses `@lobechat/observability-otel` for distributed tracing:

- Memory extraction operations
- LLM inference calls
- Database queries
- Agent evaluations

#### Configuration

OpenTelemetry configuration is currently internal. For custom exporters, modify:

```typescript
// src/server/services/memory/userMemory/extract.ts
import { SpanStatusCode } from '@lobechat/observability-otel/api';
import { attributesCommon } from '@lobechat/observability-otel/node';
```

## Application Logging

### Server Logs

LobeHub uses Next.js logging, which outputs to stdout/stderr.

#### Log Levels

- **Info**: Normal operations (startup, config loading)
- **Warn**: Deprecation notices, non-critical issues
- **Error**: Failures, exceptions

#### Viewing Logs

**Docker:**
```bash
docker logs lobehub-container
```

**Docker Compose:**
```bash
docker-compose logs -f app
```

**Node.js:**
```bash
npm run start 2>&1 | tee app.log
```

### Client Telemetry

Client-side telemetry is controlled by user preferences:

```typescript
// User setting: settings.general.telemetry
// Default: undefined (prompt user on first use)
// Values: true (enabled), false (disabled)
```

When enabled, tracks:
- Feature usage (discover agents, tool calls)
- Error events
- Performance metrics

## Performance Monitoring

### Database Performance

#### Connection Pooling

For production deployments, use connection pooling:

**Neon (Built-in):**
```bash
DATABASE_URL=postgres://user:pass@pooler.neon.tech/dbname
DATABASE_DRIVER=neon
```

**PgBouncer (Self-hosted):**
```bash
DATABASE_URL=postgres://user:pass@pgbouncer:6432/dbname
DATABASE_DRIVER=node
```

#### Query Monitoring

Monitor slow queries in PostgreSQL:

```sql
-- Enable slow query logging
ALTER DATABASE lobehub SET log_min_duration_statement = 1000; -- Log queries > 1s

-- View slow queries
SELECT query, calls, total_time, mean_time 
FROM pg_stat_statements 
ORDER BY mean_time DESC 
LIMIT 20;
```

### Redis Performance

When using Redis for sessions/caching:

**Monitor Redis:**
```bash
redis-cli INFO stats
redis-cli SLOWLOG GET 10
```

**Key Metrics:**
- Hit rate: `keyspace_hits / (keyspace_hits + keyspace_misses)`
- Memory usage: `used_memory_human`
- Connected clients: `connected_clients`

### API Rate Limiting

Monitor LLM API usage to avoid rate limits:

- Track requests per minute/hour
- Monitor token usage (Langfuse provides this)
- Set up alerts for quota approaching limits

## Error Tracking

### Sentry Integration (Optional)

For production error tracking, integrate Sentry:

1. Install Sentry:
```bash
pnpm add @sentry/nextjs
```

2. Configure `sentry.client.config.ts`:
```typescript
import * as Sentry from '@sentry/nextjs';

Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  environment: process.env.NODE_ENV,
});
```

3. Set environment variable:
```bash
NEXT_PUBLIC_SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
```

## Agent Evaluation Telemetry

LobeHub includes built-in agent evaluation with telemetry:

```typescript
// Evaluation metrics tracked:
{
  completionReason: string,  // stop, error, timeout
  duration: number,          // execution time (ms)
  tokens: number,            // total tokens used
  cost: number,              // estimated cost
  llmCalls: number,          // number of LLM invocations
  toolCalls: number,         // number of tool invocations
  steps: number,             // reasoning steps
  errorMessage?: string,     // if failed
  errorDetail?: string       // detailed error info
}
```

Access via agent evaluation UI or database tables:
- `agent_eval_runs` - Evaluation run metadata
- `agent_eval_run_topics` - Individual test results

## Security & Privacy

<Warning>
  **Privacy Considerations:**
  - User data in traces may include conversation content
  - Ensure Langfuse/observability backend is secured
  - Comply with data retention policies (GDPR, etc.)
  - Allow users to opt-out of telemetry
  - Do not log sensitive information (API keys, passwords)
  - Use encryption for data in transit and at rest
</Warning>

## Debugging Tips

### Enable Verbose Logging

Set Node.js debug environment:
```bash
DEBUG=* npm run start
```

### Database Query Logging

Enable Drizzle query logging:
```typescript
// In database config
const db = drizzle(client, { 
  logger: true  // Logs all SQL queries
});
```

### Trace Specific Requests

Use trace IDs in logs to correlate:
- Client request → Server handler → Database query → LLM call
- Look for `trace_id` in messages table
