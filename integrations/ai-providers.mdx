LobeHub supports a comprehensive ecosystem of AI model providers, giving you the flexibility to choose the best models for your specific needs. Whether you prefer commercial APIs, open-source models, or local deployments, LobeHub seamlessly integrates with your preferred infrastructure.

## Why Multi-Provider Support?

LobeHub's multi-provider architecture offers several key advantages:

- **Unified Intelligence**: Access any model and any modality from a single interface
- **Cost Optimization**: Switch between providers to optimize for performance and budget
- **Redundancy**: Avoid vendor lock-in and maintain service continuity
- **Flexibility**: Mix and match models for different agents and use cases

## Adding a Provider

### Via Settings Interface

1. Open LobeHub Settings
2. Navigate to **Language Model** section
3. Click **Add Model Provider**
4. Select your provider from the list
5. Enter required credentials (API key, endpoint, etc.)
6. Test the connection
7. Save configuration

### Via Environment Variables

For self-hosted deployments, configure providers using environment variables:

```bash
# OpenAI
OPENAI_API_KEY=sk-xxxxxx
OPENAI_PROXY_URL=https://api.openai.com/v1

# Anthropic
ANTHROPIC_API_KEY=sk-ant-xxxxx

# Google
GOOGLE_API_KEY=xxxxx

# Azure OpenAI
AZURE_API_KEY=xxxxx
AZURE_ENDPOINT=https://your-resource.openai.azure.com

# Local Ollama
OLLAMA_PROXY_URL=http://localhost:11434
```

See the complete [Environment Variables documentation](https://lobehub.com/docs/self-hosting/environment-variables) for all provider options.

## Model Selection

Once providers are configured, you can select specific models:

1. **Per-Agent Configuration**: Assign different models to different agents
2. **Session Override**: Change models mid-conversation
3. **Default Model**: Set a default for new conversations

## Custom Model Lists

Control which models appear in your LobeHub instance:

```bash
# Add custom models
OPENAI_MODEL_LIST=+custom-model-name

# Hide specific models
OPENAI_MODEL_LIST=-gpt-3.5-turbo

# Rename models for display
OPENAI_MODEL_LIST=model_name=Custom Display Name

# Combine operations
OPENAI_MODEL_LIST=+custom-model,-gpt-3.5-turbo,gpt-4=GPT-4
```

## Best Practices

- **Start Small**: Begin with one or two providers, then expand as needed
- **API Key Security**: Never commit API keys to version control
- **Cost Monitoring**: Track usage across providers to optimize costs
- **Fallback Strategy**: Configure multiple providers for redundancy
- **Local First**: Use local models (Ollama) for privacy-sensitive work
- **Test Connections**: Verify each provider configuration before deploying

## Troubleshooting

### Connection Errors

- Verify API key is correct and active
- Check base URL/endpoint configuration
- Ensure network connectivity to provider
- Review firewall and proxy settings

### Model Not Available

- Confirm model is supported by your provider tier
- Check for region restrictions
- Verify model name spelling

### Rate Limits

- Review provider's rate limit documentation
- Implement request queuing
- Consider upgrading provider tier
- Use multiple providers for load distribution
